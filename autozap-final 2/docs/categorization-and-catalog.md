# Категоризация и публичный каталог

## Структура каталога

Публичный каталог строго соблюдает цепочку:

```text
Главная
↓
Категория
↓
Подкатегория
↓
Товар
```

Главная страница показывает только основные категории.

Страница категории показывает только подкатегории выбранной категории.

Страница подкатегории показывает только товары выбранной подкатегории.

Карточка товара показывает только один товар.

## Маршруты

- `/` — главная, категории.
- `/catalog/podveska` — категория, только подкатегории.
- `/catalog/podveska/amortizatory` — подкатегория, только товары.
- `/catalog/podveska/amortizatory/a-00002-antenna-gs-2045` — карточка товара.
- `/catalog/ves-assortiment` — категория "Весь ассортимент".
- `/catalog/ves-assortiment/vse-tovary` — полный список товаров без нарушения структуры "категория → подкатегория → товары".

## Правила категоризации

Стартовые подкатегории и правила находятся в:

- `src/config/catalog-taxonomy.ts`
- `db/seeds/002_taxonomy_rules.sql`

Правила хранятся в таблице `categorization_rules` и редактируются через будущую админ-панель.

Каждое правило содержит:

- `pattern` — шаблон;
- `match_type` — тип совпадения;
- `category_id`;
- `subcategory_id`;
- `priority`;
- `is_active`.

Чем меньше `priority`, тем раньше применяется правило.

## Очередь проверки

Если товар:

- не имеет названия после кода;
- не попал ни под одно правило категоризации;
- требует ручной проверки по другой причине,

он получает статус `needs_review`, а запись добавляется в `review_queue`.

В `review_queue` сохраняется причина и, если удалось, предлагаемая категория/подкатегория.

## Обучение правил

Сервис `applyManualCategorizationCorrection()` выполняет ручное исправление администратора:

1. назначает товару категорию и подкатегорию;
2. закрывает запись `review_queue`;
3. создает новое правило в `categorization_rules`;
4. пишет действие в `audit_logs`.

Если администратор не передал шаблон правила вручную, система берет первые значимые слова из названия товара.

## Проверка покрытия правил

```bash
pnpm categorization:check data/import-samples/catalog.xls
```

Команда не пишет в базу данных, а показывает, сколько строк покрывают стартовые правила.
