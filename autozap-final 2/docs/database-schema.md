# Схема базы данных

База данных проектируется под версионный каталог. Новый Excel не меняет сайт сразу: сначала создается черновая версия каталога, затем администратор публикует ее после проверки отчета.

## Основные группы таблиц

### Админка и безопасность

- `admin_users` — учетные записи администраторов, пароль хранится только как хэш.
- `admin_sessions` — серверные сессии для доступа в `/admin`.
- `audit_logs` — журнал действий администратора.

### Контент и настройки

- `contacts` — единая запись с названием, телефоном, почтой, адресом и координатами.
- `business_hours` — график по дням недели для расчета статуса работы.
- `assets` — логотипы, favicon, OG-изображения, фото магазина, иконки категорий, изображение вакансии.
- `vacancies` — вакансии, на запуске используется продавец-консультант.
- `site_settings` — редактируемые настройки сайта.

### Каталог

- `categories` — восемь верхнеуровневых категорий, включая `Весь ассортимент`.
- `subcategories` — подкатегории внутри конкретной категории.
- `catalog_versions` — версии каталога: `draft`, `active`, `archived`, `rolled_back`.
- `products` — товары внутри конкретной версии каталога.

В `products` сохраняются `stock_quantity` и `stock_sum` как технические поля импорта, но публичный интерфейс их не показывает.

### Импорт Excel

- `import_batches` — загрузки Excel и общий отчет.
- `import_rows` — построчный результат разбора.
- `import_errors` — ошибки по строкам и полям.
- `review_queue` — товары, требующие ручной проверки.

### Поиск и категоризация

- `categorization_rules` — правила автоматического назначения категории и подкатегории.
- `synonyms` — редактируемый словарь синонимов для поиска.

## Индексы

- `products_shop_code_idx` — быстрый поиск по внутреннему коду.
- `products_search_trgm_idx` — fallback-поиск PostgreSQL `pg_trgm`.
- `products_name_trgm_idx` — поиск по названию в админке.
- индексы по версиям, статусам, категориям и датам логов.

## Расширения PostgreSQL

- `pgcrypto` — UUID.
- `pg_trgm` — резервный поиск и админ-поиск.
- `unaccent` — нормализация текста для будущего поиска.
