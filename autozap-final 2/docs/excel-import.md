# Excel-импорт

Импорт не публикует каталог автоматически. Сначала создается draft-версия, показывается отчет, затем администратор вручную нажимает `Опубликовать изменения`.

## Что поддерживается

- `.xls` и `.xlsx`;
- проверка расширения, MIME-типа и размера файла;
- определение листов и колонок;
- разделение внутреннего кода магазина и названия;
- проверка цены;
- автоматическая категоризация;
- очередь ручной проверки;
- публикация draft-версии;
- отмена draft-импорта;
- запись действий в `audit_logs`.

## Структура предоставленного Excel

В исходном каталоге обнаружен лист `TDSheet`.

Колонки:

- `A` — сырой текст номенклатуры: код магазина + название товара;
- `B` — остаток, техническое поле;
- `C` — цена;
- `D` — сумма, техническое поле.

Остаток и сумма сохраняются как технические данные импорта, но публично на сайте не показываются.

## Разбор строки

Пример:

```text
А-00002 Антенна GS 2045
```

Результат:

```json
{
  "shopCode": "А-00002",
  "name": "Антенна GS 2045"
}
```

Парсер берет первый токен формата `буквы-дефис-цифры` как внутренний код, а весь оставшийся текст как название. Если код есть, но названия нет, строка попадает в `review_queue`.

## Отчет импорта

В отчете показываются:

- добавлено;
- обновлено;
- архивировано;
- ошибки;
- требует проверки;
- список ошибок по строкам.

Публикация запрещена, если предварительный анализ не завершен или draft уже опубликован/отменен.

## Проверка без записи в базу

```bash
pnpm import:check data/import-samples/catalog.xls
```

## Ошибки, которые обрабатываются

- пустой файл;
- неподдерживаемое расширение;
- слишком большой файл;
- пустая строка;
- техническая строка до начала номенклатуры;
- итоговая строка;
- отсутствует внутренний код;
- отсутствует название после кода;
- отсутствует цена;
- цена не является числом;
- цена равна нулю;
- цена меньше нуля.
