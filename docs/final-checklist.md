## HTTP-проверки локального production-сервера

Успешно:

- `/` возвращает `200 OK`;
- `/search` возвращает `200 OK`;
- `/robots.txt` возвращает `200 OK`;
- `/sitemap.xml` возвращает `200 OK`;
- `/admin` без сессии перенаправляет на `/admin/login`;
- `/admin/login` возвращает `200 OK`;
- `/api/admin/session` без сессии возвращает `401 Unauthorized`;
- `/admin/backups` и `/admin/backups/export` без сессии перенаправляют на login;
- `/admin` и `/api/*` закрыты от индексации через `X-Robots-Tag: noindex, nofollow`;
- robots закрывает `/admin` и `/api`;
- главная содержит Open Graph и JSON-LD `AutoPartsStore`.

## Что не удалось проверить в текущем окружении

- Docker Compose не запускался: Docker CLI недоступен в окружении проверки.
- Полный runtime с PostgreSQL и Meilisearch не проверялся: нет доступных `docker` и `psql`.
- UI-сценарии в браузере, мобильные скриншоты и Lighthouse не проверялись: браузерный инструмент и Playwright/Lighthouse недоступны.
- `/catalog/...` и `/api/search` на локальном сервере без БД возвращают ошибку подключения к данным; с поднятыми PostgreSQL и Meilisearch это закрывается обычным запуском инфраструктуры.
- Главная без БД открывается с fallback-данными, но ждет таймаут подключения к базе. В production база должна быть доступна постоянно.
